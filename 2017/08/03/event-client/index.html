事件代理（事件委托）

利用事件冒泡，只指定一个事件处理程序，就可以管理某一类型的所有事件

优点

  • 减少DOM操作的次数
  • 减少内存空间的占用

原理

冒泡：从最深的节点逐步向上传播事件

运用

  • 子节点实现相同的功能：直接在父元素上绑定响应事件，发生事件是冒泡到父元素上响应

  • 不同子节点对事件作出不同反应：event.target返回事件的目标节点，对此节点做不同反应
