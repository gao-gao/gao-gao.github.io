元素显示方式display；定位position: static relative absolute fixed; 浮动float 及其在文档流中的位置

元素显示方式display

block

  • 默认宽高为父元素宽高
  • 可设置宽高
  • 换行显示
    默认为block的元素：<div>, <p>, <h1> ~ <h6>, <ul>, <form>

    inline

  • 默认宽度为内容宽度
  • 不可设置宽高
  • 同行显示（元素内部可换行）
  • 默认为inline的元素：<span>, <a>, <label>, <cite>, <em>

    inline-block

  • 默认宽度为内容宽度
  • 可设置宽高
  • 同行显示
  • 整块换行
  • 默认为inline-block的元素：<input>, <textarea>, <select>, <button>

    none

  • 设置元素不显示（不占位）
  • visibility：hidden 不显示但占位

定位机制

标准文档流

从上到下，从左到右的输出文档内容。它由块级（block）元素和行级元素组成，且它们都是盒子模型。

浮动（float）

绝对定位

定位形式position

静态定位（static）

  • 标准文档流
  • position的默认值，当宁外设置position值时，会覆盖掉此默认值
  • 无top、right、bottom、left值

相对定位（relative）

  • 仍在文档流中并按照文档流中的顺序进行排列

  • 参照物为元素本身的位置

  • 其他元素不会占用相对定位元素原始的位置

  • 相对定位元素可能会与其他元素重叠

绝对定位（absolute、fixed）

  • absolute：
  1. 建立以包含块为基准的定位，其随即拥有偏移属性和 z-index 属性
  2. 默认宽度为内容宽度
  3. 脱离文档流
  4. 参照物为第一个定位祖先或根元素html
  5. 无定位祖先元素时，不用top,left,right,bottom属性或者是auto值，原来在什么地方，还是在什么地方，只是漂浮起来了，通过margin值来设置精确定位。
  6. 绝对定位的拉伸可以和height，width相互替代
  7. 可以取代fixed实现遮罩效果（移动端使用优于fixed）
  • fixed：
    默认宽度为内容宽度
    脱离文档流
    参照物为视窗（浏览器）
    设置定位元素相对于参照物的距离，设置同一轴上的两个参数时，元素会被拉伸

参数 z-index

  • 使用时先确认是否设置了relative,absolute,fixed三种中的一种

  • 设置Z轴上的排序，数字越大，越在表层

  • 父容器的值权重大于子元素的值

浮动

float：left|right

实现块级元素同行显示并存在于文档流中，浮动影响文档流中下一个紧邻的元素

  • 默认宽度为内容宽度

  • 半脱离文档流

对于元素是脱离文档流，但是对于内容是在文档流之中（即元素重叠，但内容不重叠）

  • 按照值的方向一直移动

浮动元素及其父元素宽高

浮动元素脱离文档流，但元素中的内容在文档流中

浮动元素宽高不影响其父元素宽高

  • 父元素宽高若不自行设置，即使浮动的子元素都设置了宽高，父元素仍会塌陷（高度为0），嵌套的元素则不会正确排列。

清除浮动

  • 浮动元素后添加空白标签，并设置其样式clear:both

  • overflow:auto||hidden，前者IE6会给元素添加滚动条，最好用后者
    在具有浮动元素的父元素中设置一个自动扩展的高度（块级元素宽度默认为100%不需设置）
    此法相当于给父元素设置宽高

  • clearfix：利用“:before”和“:after”伪元素，在浮动元素的父容器前面和后面创建隐藏元素,并加上zoom属性触发父元素的hasLayout属性
    通常用一个类表示需要设置清除浮动的元素，通过类名为元素添加样式
