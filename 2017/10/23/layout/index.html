深入了解各种布局原理

学习前端以来，接触过很多种布局方式，也学习过如何用多种方法实现一种布局，但是当接触到设计图时，还是不能真正地将所学的知识融会贯通，往往会面临下面两种问题：

  • 需求中的布局有多种方式实现，不知道怎么根据当前需求去选择最好的哪一种
  • 要求实现的布局没有接触过，完全不知道从哪里下手

对此，我进行了反思和总结，发现自己在学习过程中只是粗浅地去了解某种布局的实现方式，而不是去探究为何这些属性的设置可以实现这个布局，当我去探究时却又发现对于很多属性和值的特性只是停留在字面上的记忆，而没有基于文档流去理解，导致在越来越深入和扩展的学习中，越来越迷糊。

所以想要真正拥有解决问题的能力，而不是一味地模仿别人的做法，我需要对CSS的一些属性和值进行一次深入的学习。

display：显示方式

display属性用来设置元素的显示，有多个值，不同的值对应了元素在页面中不同的显示特性。每个元素都有一个初始的默认值（根据默认值将元素分为了块级元素、行内元素、行内块级元素等），可以不设置值使用其默认值，也可以根据需求设置取代默认值。

block

  • 默认占满整行，不设置高度时被内容撑起
  • 可设置宽高
  • 换行显示

inline

  • 宽度为内容宽度
  • 不可设置宽高
  • 同行显示，可换行（指的是内容占满整行放不下的时候）

inline-block

  • 宽度默认为内容宽度
  • 可设置宽高
  • 同行显示，换行时是整块进行换行

none

  • 不显示，且在文档流中不占位置；和visiblity不同，visibility：hidden也是不显示，但是依旧占据原本的位置

table类：拥有table的属性

  • table: 作为块级表格显示，表格前后有换行符
  • table-cell: 作为一个单元格显示

flex

  • 作为弹性伸缩盒显示，其具体参数见：flexbox布局

position：定位方式

文档中的元素默认定位方式是static，即根据元素的特性从左到右、从上到下，渲染出整个页面。设置定位（除去static）后，通常可以用top,bottom,right,left四个值设置元素的位置，用z-index设置在z轴上的位置（即上下摆放的顺序）。

relative

  • 仍在文档流中，定位后原本的位置不会被占据（即移动之后不会对下列元素产生影响）
  • 默认宽度为由元素性质决定
  • 显示层级高于正常文档流
  • 参照物为自身原本的位置
  • 只用设置同一轴上的一个方位的定位
  • 通常与absolute一起使用，设置在父元素上

absolute

  • 完全脱离文档流
  • 宽度为内容宽度
  • 显示层级高于正常文档流
  • 参照物为第一个定位祖先元素或者是根元素
  • 同时设置同一轴上的两个方向值时，会将元素拉伸(在不设置宽高时）;优先选取left和top值（设置宽高时）

fixed

  • 完全脱离文档流,不占据空间
  • 宽度为内容宽度
  • 显示层级高于正常文档流
  • 参照物为浏览器窗口
  • 同时设置同一轴上的两个方向值时，会将元素拉伸(在不设置宽高时）;优先选取left和top值（设置宽高时）

z-index

只有设置了relative，absolute,fixed这三种定位的元素通过z-index设置在z轴上的定位，数字越大，层级越高

  • z-index栈：当多个元素设置z-index后，在判断其堆叠顺序时，优先比较最外层元素的z-index值

float

特性

  • 半脱离文档流，对元素脱离文档流，对内容不脱离文档流
  • 默认宽度为内容宽度
  • 设置left和right值时，一直向左或右移动知道碰到浏览器边界或文档流中的其他内容

清除浮动方法

见文章布局基础知识尾部

固定顶栏

  • body：padding-top:50px
  • 固定顶栏样式：position:fixed; top:0; left:0; height:50px;

设置body的padding和遮罩元素的宽度相等原因：设置了fixed的元素完全脱离文档流，就会将后面的内容遮住，所以需要在正常文档流中将遮罩的高度预留出来。

透明遮罩

  • mask样式：

设置宽高为100%占满整个浏览器(也可以不设置宽高，同时设置上下左右四个方向的值为0)，为遮罩加一个透明度，即可呈现遮罩效果。但是需要注意的一点是：被遮罩遮住的地方都只是可读，而不可操作（比如说链接的点击，hover效果等无法呈现）

三行中间行自适应

即头部和尾部固定高度且固定在浏览器窗口上方和下方的位置，中间一栏自适应高度，通常做法如下，设置中间部分为绝对定位且设置溢出时以滚动条形式显示,这样中间就是自适应的高度了，

注意：因为中间部分设置了绝对定位使中间部分完全脱离文档流，中间部分会直接遮住尾部区域，当内容高度较大时，滚动滚动条会遮盖住头部区域，所以必须对头部和尾部进行设置定位以消除这种影响。

有两种方法可以解决：为头部尾部同时设置（案例代码中演示的是absolute）

  • 设置fixed
  • 设置absolute

当需求中的三行自适应是相对于整个浏览器窗口时，两种方法都是正确的；但是当相对的只是页面中的一个小范围的区域时，选择absolute结合relative来实现。

  • 设置overflow：auto，让溢出内容以滚动条的形式显示，这样就不会扩展页面中视觉上的高度。

水平居中布局

当宽度确定的时候，通常采用：margin：0 auto方法，当宽度不确定时，使用下面这些方法

inline-block + text-align

为需要居中的元素的设置inline-block(利用其默认宽度为内容宽度的特性)，让其变成行内块级元素，并在其父元素上设置text-align:center，就可以让需要居中的元素居中

  • 利用行内元素text-align属性实现居中

缺点：IE7，8不支持inline-block,需要做兼容

table+margin

table属性表现上像block,但是其默认宽度是内容宽度，所以结合margin：0 auto，直接在需要居中的元素上设置这两个样式

  • 利用块级元素margin：0 auto和table元素默认宽度为内容宽度实现居中

缺点：IE8,7不支持table属性，但是支持table结构，可直接使用table标签实现

absolute+transform

通过设置绝对定位，让元素处于父元素宽度的50%处，此时，元素的中心点并不在父元素的50%处，所以通过transform：translateX(-50%)将其向左移动元素的一般宽度。

  • 利用translateX(-50%)百分比的大小是相对于元素自身宽度的特性

缺点：transform是CSS3中新增的属性，兼容性差，实现兼容也比较困难。且要注意的一点是：absolute完全脱离了文档流，所以对于页面中后续其他处于正常文档流中的会有遮盖，所以要对其父元素的高度进行处理。

flex

通过对父元素设置display: flex; justify-content: center;,这样父元素就变成了flex容器，子元素就是flex-item，便可直接设置子元素的对齐方式为居中

  • 利用flex布局

缺点：低版本IE如（6，7）不支持

垂直居中布局

设置垂直居中，很自然想到一个属性vertical-align：middle，但是这个属性适用于内联及table-cell元素，这个属性的解释是：将支持vertical-align特性的对象的内容与对象中部对齐。

table-cell+vertical-align

为需要设置垂直居中的元素的父元素设置table-cell让其成为拥有垂直居中特性的对象，让其子元素成为其内容，并结合vertical-align让元素垂直居中

  • 利用vertilcal特性

兼容性比较好，IE8一样，兼容6，7时使用table结构

absolute+transform

flex

水平垂直居中布局

当宽高都不定时，结合前面的方法一起使用。

inline-block设置水平，table-cell设置垂直

absolute+transform同时设置两个方向上

flex同时设置两个方向上

多列布局

定宽+自适应(定宽:left)

  • float+margin

使用margin：因为左边设置float之后，左边区域就半脱离文档流了，所以在正常区域中设置一个margin值，就不会和左边部分重合。

缺点：一个浮动一个不浮动在IE下面存在3px的缩进bug

  • float+float+fix

利用relative提升层级

  • float+overflow

BFC规则 ,将不浮动的block for content

  • table

根据内容排版，一列定宽占宽度。优先根据布局
：table-layout：fixed。不能设置margin值

  • flex

定宽部分设置宽度，剩余部分平分剩余宽度，可以设margin值。一般用于小范围的布局，大范围的布局会影响页面的性能。

定宽+定宽+自适应

  • float+float+overflow

不定宽+自适应

也就是说，宽度由内容决定，那么定宽区域的宽度就是未知的。所以，在前面的定宽的做法中，使用到了左边区域的宽度值的做法都不可取。

  • float+overflow

  • table

去掉前面的table-layout：fixed，让其根据内容定宽高。

  • flex

等分布局

  • float+宽为百分比，但如果需要设置间距则会出现问题

解决方案：父容器将增加间距的剪掉，子容器设置padding。

缺点：结构和样式有耦合性

  • table
    重布局，不设置宽度子元素将平分宽度，间距通过设置每一个子元素的padding值来设置，所以要将设置多了的那个宽度剪掉，通过添加一个parent-fix来修正

  • flex
    将剩余空间进行分配，所以不能剪掉超出部分，而是要将间距进行正确设置，确保剩余空间是正确的

多列布局中设置列看起来等高

通过设置padding-bottom值无限大来实现，并用margin-bottom的负值结合overflow:hidden将不需要的高度隐去。实现两列高度看起来一样，其实是背景一样，实际上高度还是不一样的，只不过能显示出不一样的那一部分已经被overflow隐藏掉了。
