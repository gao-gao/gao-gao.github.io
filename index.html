由 ECMA-262 定义的 ECMAScript 与 Web 浏览器没有依赖关系。实际上，这门语言本身并不包含输 入和输出定义。ECMA-262 定义的只是这门语言的基础，而在此基础之上可以构建更完善的脚本语言。 我们常见的 Web 浏览器只是 ECMAScript 实现可能的宿主环境之一。宿主环境不仅提供基本的 ECMAScript 实现，同时也会提供该语言的扩展，以便语言与环境之间对接交互。而这些扩展——如 DOM，则利用 ECMAScript 的核心类型和语法提供更多更具体的功能，以便实现针对环境的操作。其他 宿主环境包括 Node（一种服务端 JavaScript 平台）和 Adobe Flash。

文档对象模型（DOM，Document Object Model）是针对 XML 但经过扩展用于 HTML 的应用程序编 程接口（API，Application Programming Interface）。

async：可选。表示应该立即下载脚本，但不应妨碍页面中的其他操作，比如下载其他资源或等待加载其他脚本。只对外部脚本文件有效。
defer：可选。表示脚本可以延迟到文档完全被解析和显示之后再执行。只对外部脚本文件有效。IE7 及更早版本对嵌入脚本也支持这个属性。

js是弱语言（解释型语言），所以可以改变变量的类型

数据类型

五种简单数据类型：boolean number string undefined null
一种复杂数据类型：object

操作符typeof并不是一个函数，所以可以加（）也可以不加:boolean string number undefinded function object

undefined

undefined: 变量被声明未初始化，默认初始值是undefined；变量未声明，依旧会返回undefined值，但严格模式下使用typeof会导致错误。所以编码时一般显式声明变量会更好，使用typeof的时候可以区分变量是未被声明还是未初始化。

显式声明和隐式声明？

null

null：空对象指针。如果打算声明一个变量用来保存对象，可以将其初始化为null,这样检查变量的时候就知道保存了一个值为null的对象的引用，可以区分undefined。

number

进制
  • 八进制：第一位必须是0，如果字面值超出了范围，将忽略前导0，作为十进制解析（在严格模式下无效，会抛出错误）
  • 十进制：算数计算时，一律转换为十进制进行计算
  • 十六进制：以0x开头
浮点数
  • 小数点前可以没有数字，如”.1”，就是0.1
  • 小数点后没有数字或者是数字0，则作为整数解析保存，因为浮点数值需要的内存空间是整数的两倍。
  • 极大或者极小的数值用e表示，浮点数值的最高精度是17位小数
  • 浮点数进行计算时会产生舍入误差的问题
数值范围

如果某次计算的结果超过了js的数值范围，数值会转换为infinity值，如果是负值，则是-infinity。可以用isFinite()函数进行判断

NaN

用于表示一个本要返回数值的操作数未返回数值的情况，比如任何数字除以0都会导致错误

  • isNaN函数：接收一个参数之后先尝试着将参数转换为数值，然后进行判断是不是数值，是返回false，不是返回true
数值转换
  • Number() 按照类型转换将空字符串，string，布尔值转换为响应的数字
  • parseInt() 如果第一个非空字符不是数字或者负号，返回NaN。转换字符串时看其是否符合字符格式，会忽略不符合数值格式的字符，例如小数点字母之类的。为避免错误的解析，通常要指定解析的进制的基数
  • parseFloat() 和parseInt()解析字符方式相似，不过在遇到第一个小数点时会识别，第二个小数点（包括第二个小数点）之后忽略

string

toString()
  • number boolean object string都有此方法，但null和undefined没有
  • 有参数时，可以传入指定的进制的基数将数字转换为相应的格式，不传参数默认十进制
String()
  • 可以转换null和undefined为相应的字符串

object

一组数据和功能的结合，object是所有他的实例的基础，其具有的方法和属性同样存在于其创建的实例中。

  • 前置：先递增递减再运算

  • 后置：先运算再递增递减
